## Suspected-but-not-found

The task brief asked me to specifically check whether "opening cards / solving groups can incorrectly trigger tier transitions". I traced the store state machine (`src/game/state/store.ts`): tier cursors only advance on `submit → win` in the matching pool, `startDaily` and menu navigation do not mutate them, and there is no code path that elevates difficulty from play interactions. **Not found, nothing to fix there.**

The "stale state on reload escalates difficulty" suspicion was also checked: `tryResumeSession` defensively clears the saved session if the puzzle id is not in the pool or if the `active.puzzleId` does not match, so a fresh install cannot inherit a higher-tier session. **Not found.**
