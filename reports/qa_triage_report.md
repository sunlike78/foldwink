Findings were dumped to `test/qa-dump.json` via `scripts/dump-qa-xlsx.py` (stdlib-only xlsx parser) and then classified by hand against current source.

## Methodology

Only rows with non-empty tester notes (or a clearly actionable empty-status row) were triaged. Empty-status rows in PROGRESSION / SOUND / SHARE-CARD / ITCH / VISUAL / EDGE-CASES sheets are **not** bugs — they are test slots the tester did not execute this pass and they are re-tested by the automated agents in `tests/e2e/`. Rows describing purely external console noise are classified TEST_ARTIFACT and not touched.

Classification taxonomy (as requested in the task brief):

| Bucket            | Meaning                                                 |
| ----------------- | ------------------------------------------------------- |
| CONFIRMED_BUG     | Reproducible mismatch between expected and actual       |
| LIKELY_BUG        | Strong signal of broken behavior, reproduction probable |
| PRODUCT_AMBIGUITY | Intentional behavior not explained by UI copy           |
| UX_IMPROVEMENT    | Not broken, but confusing, weak, or rough               |
| TEST_ARTIFACT     | Stale state / environment / extension / third-party     |
| NOT_REPRODUCIBLE  | Could not reproduce after reasonable attempts           |
